üß† 1. localStorage
=> Persists even after the browser is closed
=> Data is stored indefinitely until cleared by the user or your code
=> Stores data as key-value pairs
=> Limit: ~5MB
=> Only accessible via JavaScript (not sent with requests)
=> You need to keep data long-term (e.g., user preferences, theme selection).
example.
    localStorage.setItem("theme", "dark");
    //console.log(localStorage.getItem("theme")); // "dark"
    Common uses::	Preferences, themes


üïí 2. sessionStorage
    => Cleared when the tab or window is closed
    => Same API as localStorage
    => Limit: ~5MB
    => Useful for temporary data (like form state) 
    => Data should only last for the duration of the tab session.
    example.
    sessionStorage.setItem("tempScore", "100");
    //console.log(sessionStorage.getItem("tempScore")); // "100" 
    Common uses::	Temp forms/data
                            
            
üç™ 3. cookies
    => Small pieces of data (limit ~4KB per cookie)
    => Can be set to expire at a specific time
    => Automatically sent with every HTTP request
    => Accessible from both JavaScript and server-side
    => Used for authentication, tracking, and session handling
    => You need to send data to the server on every request (e.g., auth tokens).
    example.
    document.cookie = "username=John; expires=Fri, 1 May 2025 12:00:00 UTC; path=/";
    Common uses::	Auth, sessions

                            => Screen size

                            => srcset gives the browser multiple image sources with their widths (w = pixels).

                            => The browser picks the best image based on current screen width + resolution.    
            
                    
                
                    
                        üåü // What is Shadow DOM?
                            The Shadow DOM is a browser feature that lets you attach a hidden, separate DOM tree to an element ‚Äî one that is isolated from the main DOM.
                            //OR
                        // The Shadow DOM is a part of the Web Components standard that allows you to create encapsulated DOM trees ‚Äî meaning elements can have their own isolated structure and styles.   
                            => It's like giving an element its own private part of the page that:
                            => Has its own elements and styles
                            => Is not affected by outer page styles
                            => Doesn‚Äôt mess up your global CSS or scripts
                            It‚Äôs like giving a component a private, hidden DOM that:
            
                            => Has its own HTML elements
                            => Has its own CSS
                            => Doesn‚Äôt clash with the rest of the page
            
            
                            üß± Why Shadow DOM Exists
                            In regular HTML:
                            => Styles and scripts are global.  
                            => Components can interfere with each other (CSS leaks).               
                            => With Shadow DOM:                
                            => You can build self-contained components that work anywhere, without breaking other code or styles.

                  // What is the difference between relative, absolute, fixed, and sticky positioning?
              
                  // 1. Relative
                      => The element is positioned relative to its normal position.
                      => It moves, but doesn‚Äôt affect other elements.
                      => The "gap" it leaves behind stays.

                  //2. Absolute
                      => The element is positioned relative to the nearest positioned ancestor (relative, absolute, or fixed).
                      => If no positioned ancestor is found, it positions relative to the < html> (viewport).
                      => It‚Äôs completely removed from the normal document flow.    
              
                  //3. Fixed
                      => The element is positioned relative to the browser window (viewport).
                      => Does not move when you scroll!
                      => Always stays at the same place.

                  // 4. Sticky
                      => It‚Äôs a hybrid of relative + fixed.
                      => The element acts like relative until a scroll threshold is reached, then sticks like fixed.                
                      => Useful for sticky headers, sidebars, etc.    
              
          
              
                  // 1. px (pixels)
                      => Fixed size unit.
                      => Not responsive ‚Äî size stays the same no matter parent or root.
                      => 1px = 1 pixel on the screen.

                  //2. em (relative to parent)
                      => Size is relative to the parent‚Äôs font-size.
                      => Multiplies down the hierarchy (can cascade).
                      
                  // 3. rem (relative to root)
                      => root em. 
                      => Always relative to the < html> font-size.
                      => Consistent scaling across the page ‚Äî does not depend on parent elements.    
                  
                  => px = Absolute.
                  => em = Relative to parent.     
                  => rem = Relative to root.
          
                  
          
              
                  // px, em, rem, %, vh ‚Äî Quick Comparison

                   => Unit || Stands for || Based on || Good for || Example
                   => px || Pixels || Fixed size || Absolute measurements (e.g., borders) || font-size: 16px;
                   => em || Relative to parent || Parent‚Äôs font-size || Nested scaling (can cascade) || font-size: 2em;
                   => rem || Root em || < html> font-size || Consistent scaling across the page || font-size: 1.5rem;
                   => % || Percentage || Relative to parent element || Responsive sizing || width: 50%;
                   => vh || Viewport height || 1% of viewport‚Äôs height || Full-screen sections || height: 100vh;
